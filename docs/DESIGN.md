# 设计规范

## UI 设计原则

1. **简洁优先**：界面只展示当前任务所需的信息，避免信息过载
2. **状态清晰**：录音/处理/完成等状态必须在 UI 上有明确且一致的视觉区分
3. **操作反馈**：所有用户操作在 300ms 内给出视觉反馈

## 颜色语义

| 用途 | 含义 |
|------|------|
| 主色 | 品牌色，用于主操作按钮 |
| 红色 | 录音中（录音按钮高亮）、错误状态 |
| 绿色 | 成功、完成状态 |
| 灰色 | 禁用、次要信息 |
| 琥珀色 | 处理中（AI 运行中） |

## 会议状态的 UI 表现

| 状态 | 录音按钮 | 状态提示 |
|------|---------|---------|
| idle | 默认样式，可点击 | 无 |
| recording | 红色脉冲动画 | "录音中…" |
| processing | 禁用，加载动画 | "AI 处理中…" |
| done | 恢复默认 | 显示摘要卡片 |
| error | 恢复默认，警告图标 | 错误信息 |

## 页面导航

- **Home**：会议列表，按时间倒序，点击进入 Meeting 页
- **Meeting**：当前/历史会议详情，含转写、行动项、报告三个 Tab
- **Settings**：LLM 配置、Whisper 模型选择、音频设备选择

## 响应式

桌面应用，目标窗口最小尺寸：1024 × 680px。不需要移动端适配。

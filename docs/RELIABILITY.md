# 可靠性目标

## 核心原则

**数据不丢失**：音频文件和会议记录是用户的重要资产，任何情况下不应丢失。

## 关键路径保护

### 录音过程

- 音频数据实时写入磁盘（流式写入），不在内存中缓冲整个录音
- 应用崩溃时，已录制的音频文件必须保留
- 录音中途异常停止，`meetings.status` 更新为 `error`，保留 `audio_path`

### AI 处理过程

- Pipeline 每完成一个 Stage，立即将结果持久化到数据库
- 单个 Stage 失败不影响已完成的 Stage 结果
- 单个 Stage 超时（60s）后重试一次，仍失败则跳过，继续执行后续 Stage

### 数据库

- 使用 SQLite WAL 模式，提高并发写入安全性
- 多步数据库操作使用事务，原子性保证
- 应用启动时运行 schema 迁移，确保表结构最新

## 错误分类与处理

| 错误类型 | 处理方式 | 用户提示 |
|---------|---------|---------|
| 音频设备不可用 | 显示设备列表，引导选择 | "未找到录音设备，请检查麦克风连接" |
| Whisper 模型未找到 | 跳转 Settings 页 | "请先下载 Whisper 模型" |
| Ollama 连接失败 | 提示检查服务状态 | "无法连接到 Ollama，请确认服务已启动" |
| LLM 响应格式错误 | 存储原始文本，标记阶段 | "AI 处理部分失败，已保存原始转写" |
| 磁盘空间不足 | 停止录音 | "磁盘空间不足，录音已停止" |

## 用户数据目录

```
{app_data_dir}/
├── memo-ai.db           # SQLite 数据库
├── recordings/          # 音频文件
│   └── {meeting_id}.wav
└── logs/                # 应用日志（v1.0）
```

`app_data_dir` 由 Tauri 根据操作系统自动确定：
- Windows：`%APPDATA%\memo-ai`
- macOS：`~/Library/Application Support/memo-ai`

## 日志策略（v1.0）

- 记录级别：ERROR 和 WARN 默认开启，DEBUG 需用户手动开启
- 日志文件大小上限：10 MB，自动滚动，保留最近 3 个文件
- 日志不包含会议内容（隐私保护）

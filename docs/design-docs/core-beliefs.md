# 核心设计理念

## 1. 本地优先（Local-First）

用户数据完全存储在本机，AI 处理默认在本地完成（Whisper + Ollama）。云端 API（OpenAI）作为可选项，而非默认。

理由：隐私保护、无网络依赖、低延迟。

## 2. 会议结束即文档完成

产品的终极目标是让用户在会议结束时，所有文档（纪要、行动项、报告）已自动生成完毕，无需任何手动整理。

## 3. AI Pipeline 分阶段处理

不使用单一大 Prompt 处理所有任务，而是拆分为 6 个独立阶段，每个阶段有明确的输入/输出格式。

好处：
- 每个阶段可独立调试和优化
- 错误影响范围可控
- 支持不同阶段使用不同模型
- 便于 A/B 测试 Prompt

## 4. Rust 承担所有重型计算

音频采集、ASR 推理、LLM 调用、数据库操作均在 Rust 层完成。React 前端只负责展示和用户交互。

好处：性能、内存安全、跨平台一致性。

## 5. 结构化输出优先

所有 AI 阶段的输出尽量使用 JSON 结构，而非自由文本，以确保数据可靠解析和存储。

## 6. MVP 路径清晰

第一版只做录音 + 转写 + AI 总结三个核心功能，其余功能（知识库、问答、集成）留待后续迭代。

# 产品规格：AI 处理管道

## 功能目标

将会议录音自动转化为结构化会议文档，包括转写文本、会议纪要、行动项和完整报告。

## 用户价值

- 会议结束即获得完整会议文档
- 无需手动记录，专注会议本身
- 行动项自动提取，不遗漏任何承诺
- 通过 ACP 智能体多轮对话，灵活精炼报告和纪要，输出更符合个人/团队风格

## 功能列表

### ASR 转写

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 本地转写 | Whisper.cpp 本地推理 | MVP |
| 实时字幕 | 录音时实时显示转写 | v1.0 |
| 说话人识别 | 区分不同发言人 | v1.0 |
| 多语言支持 | 中文优先，英文次之 | MVP |

### AI 分析

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 文本清洗 | 修复 ASR 错误和断句 | MVP |
| 结构化提取 | 主题/参会人/要点/决议/风险 | MVP |
| 会议总结 | 生成会议纪要 | MVP |
| 行动项提取 | 任务/负责人/截止日期 | MVP |
| 报告生成 | Markdown 格式完整报告 | MVP |
| ACP 报告精炼 | 报告生成后，通过 ACP 智能体多轮对话调整报告 | v1.0 |
| ACP 纪要调整 | 会议总结后，通过 ACP 智能体调整纪要风格/详略 | v1.0 |
| ACP 行动项确认 | 行动项提取后，通过 ACP 智能体批量修改任务分配 | v1.0 |
| ACP 结构修正 | 结构化提取后，通过 ACP 智能体补充/修正参会人和决议 | v1.0 |

### LLM 配置

| 选项 | 默认 | 说明 |
|------|------|------|
| Ollama（本地） | 是 | 推荐 Llama3 8B 或 Mistral 7B |
| OpenAI API | 否 | 需要用户提供 API Key |

## 用户配置（Settings 页）

1. 选择 LLM 后端（Ollama / OpenAI）
2. Ollama：配置模型名称和服务地址（默认 `http://localhost:11434`）
3. OpenAI：填写 API Key 和模型名称（默认 `gpt-4o-mini`）
4. Whisper 模型大小：tiny / base / small（影响速度和准确度）
5. ACP 智能体地址（默认 `http://localhost:8333`）
6. ACP 智能体 ID（默认 `meeting-assistant`）
7. 启用 ACP 的阶段（可多选：结构提取 / 会议总结 / 行动项 / 报告）

## 处理时间预估（参考）

| 会议时长 | Whisper (base) | LLM Pipeline |
|---------|---------------|-------------|
| 30 分钟 | ~2 分钟 | ~1 分钟 |
| 60 分钟 | ~4 分钟 | ~2 分钟 |

（基于 RTX 4060 本地显卡推理）

## 错误处理

- ASR 失败：显示错误提示，保留音频文件，允许重新处理
- LLM 超时：单阶段超时 60s 后重试一次，仍失败则跳过该阶段
- JSON 解析失败：降级为原始文本存储，不影响其他阶段

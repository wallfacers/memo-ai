# 一、AI会议助手完整产品定义

产品定位：

**AI Meeting Copilot**

核心价值：

```
会议录音
→ 自动转写
→ 自动理解
→ 自动生成行动项
→ 自动生成报告
```

最终目标：

**会议结束 = 文档已经完成**

---

# 二、产品功能体系

完整产品需要 **6个模块**

```
1 会议采集
2 语音识别
3 会议理解
4 知识管理
5 AI助手
6 集成系统
```

---

# 三、模块一：会议采集系统

这是最基础的模块。

### 功能

🎙️ 录音

支持：

* 麦克风
* 系统音
* 双通道录音

例如：

```
Channel 1: 麦克风
Channel 2: 系统音
```

应用场景：

* Zoom会议
* Teams会议
* 腾讯会议
* Google Meet

实现：

```
Rust Audio Engine
+
WASAPI (Windows)
CoreAudio (macOS)
ALSA / Pulse (Linux)
```

关键能力：

* 自动降噪
* 音量均衡
* 音频分轨

---

开源：

* Whisper
* Parakeet
* FunASR

商业：

* OpenAI STT

* AssemblyAI

# 五、模块三：AI会议理解

这是 AI会议软件的核心。

输入：

```
完整会议转录
```

输出：

### 1 会议纪要

结构化：

```
会议主题
参会人
讨论要点
结论
行动项
```

---

### 2 行动项提取

AI提取：

```
TODO
负责人
截止日期
```

例子

```

完成API设计

负责人：
张三

截止日期：
3月10日
```

---

### 3 决策提取

AI识别：

```
决议
```

例如：

```
决定：
第一阶段先内部测试
```

---

### 4 会议摘要

短摘要：

```

```

长摘要：

```
完整会议纪要
```

### 会议问答

